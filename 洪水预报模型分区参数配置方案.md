# 15个流域洪水预报模型分区参数与边界条件配置方案

> 说明：本方案针对15个流域（分区）洪水预报模型，系统规定各分区在建模与预报运行时需分别设置的**核心初始参数**、**初始条件**（土壤含水量、初始地下水储量、初始基流）、**预热期**、**上边界条件**（流域面/栅格雨量、流域面/栅格潜在蒸散发过程）、**内边界条件**（上下游汇入流量过程）、**下边界条件**（流域出口流量过程）六大类要素的配置内容、技术方法与台账模板，其中面/栅格雨量与面/栅格潜在蒸散发须采用地理空间插值方法，由监测站点观测数据插值得到。文中流域名称、站点及参数取值均为示例占位，实际应用时应替换为15个流域的具体资料并据实率定。

---

## 一、总体框架

洪水预报模型按"流域—分区"两级管理，每个流域划分为若干产汇流计算单元（子流域/栅格单元），15个流域分区独立建模、独立配置、独立运行，通过统一的模型管理平台实现参数集中管理、边界条件自动驱动与预报成果集中展示。每个流域分区的配置要素及相互关系如下：

```
核心初始参数（模型结构参数）
        ↓
初始条件（土壤含水量、初始地下水储量、初始基流）→ 预热期（模拟至状态收敛）
        ↓
   ┌─────────────┬──────────────┬──────────────┐
 上边界条件      内边界条件        下边界条件
（面/栅格雨量、  （上下游汇入      （流域出口
 面/栅格潜在     流量过程）        流量过程）
 蒸散发过程）
        ↓
   洪水预报模型运算（产流—汇流—河道计算）
        ↓
     流域出口断面预报流量过程
```

## 三、核心初始参数设置

每个流域分区均需独立设置以下产流、汇流核心结构参数（以蓄满产流类概念性水文模型为例，分布式栅格模型可类比设置栅格化参数场）：

### （一）蒸发参数

| 参数 | 含义 | 设置方式 |
|---|---|---|
| K | 流域蒸发能力折算系数 | 按分区多年蒸发观测与陆面蒸发关系率定 |

### （二）产流参数

| 参数 | 含义 |
|---|---|
| WM | 流域平均张力水蓄水容量 |
| WUM、WLM、WDM | 上层、下层、深层张力水容量 |
| B | 张力水蓄水容量曲线指数（反映下垫面不均匀性） |
| C | 深层蒸发系数 |
| IM | 不透水面积比例 |

### （三）水源划分参数

| 参数 | 含义 |
|---|---|
| SM | 自由水蓄水容量 |
| EX | 自由水蓄水容量曲线指数 |
| KI | 壤中流出流系数 |
| KG | 地下水出流系数 |

### （四）汇流参数

| 参数 | 含义 |
|---|---|
| CI | 壤中流消退系数 |
| CG | 地下水消退系数 |
| CS | 河网蓄水消退系数 |
| KE、XE（或L、CR） | 河道洪水汇流（马斯京根法/滞时演算法）参数 |

**配置要求**：15个流域分区的核心参数应分别独立率定与存储，形成"一流域一参数集"，参数率定采用历史多场次洪水资料，按目标函数（确定性系数、洪峰误差、峰现时间误差等）优选，并建立参数库版本管理机制，支持后续滚动优化与参数移用（无资料分区可采用相似流域参数移用法）。

## 四、初始条件设置

初始条件用于描述预报起始时刻流域下垫面蓄水状态，是预热期模拟的起点，各流域分区需分别设置：

### （一）初始土壤含水量

- **上层初始张力水含量 W0U**：一般取上层土壤蓄水容量WUM的适当比例，或根据前期实测/再分析土壤湿度产品赋值；
- **下层初始张力水含量 W0L**：按下层蓄水容量WLM及前期降雨蒸发过程估算；
- **深层初始张力水含量 W0D**：按深层蓄水容量WDM及前期土壤水消退过程估算；
- 三层之和 **W0 = W0U + W0L + W0D**，作为流域初始土壤含水总量，其取值不应超过流域最大蓄水容量WM。

### （二）初始地下水储量

- **自由水初始蓄量 S0**：反映包气带自由水蓄存状态，一般根据预报起始时刻前的产流退水过程反算，或取多年同期均值；
- 各分区自由水初始蓄量应与该分区SM、EX参数相匹配，避免初始状态失真导致预报起涨异常。

### （三）初始基流

- **初始地下水出流（基流）QG0**：一般由预报起始时刻实测流量过程线退水段分割得到，作为地下水消退计算的起点；
- 无实测流量分区可采用前期同类水情下的基流经验值或邻近参证站基流移植后按面积比换算。

> 上述初始条件均应逐流域、逐分区单独设定，并作为预热期启动时刻（t₀）的状态初值写入模型初始状态文件，实现"一流域一初值"管理。

## 五、预热期设置

- **设置目的**：消除人为设定初始条件（土壤含水量、地下水储量、初始基流）带来的状态误差，使模型经过一段时间连续模拟后，产流、汇流各状态变量收敛至与实际流域蓄水状态相符的水平，保证正式预报期起涨点的可靠性；
- **预热期时长**：一般概念性水文模型（新安江模型、API模型、水箱模型）建议预热期30～90天，具体依流域调蓄能力（流域面积、河网密度、地下水补给周期）确定，调蓄能力强、汇流路径长的流域取上限；分布式栅格模型或涉及深层地下水动态的流域，预热期可延长至60～180天甚至更长；
- **驱动数据要求**：预热期同样需要连续、完整的降雨、蒸散发驱动数据（与上边界条件数据源一致），不得空档或以均值替代，保证状态变量连续演进；
- **收敛判定**：以预热期末流域产流量、地下水蓄量等状态变量的日际变化率趋于稳定（如变化率小于设定阈值）作为预热期结束、进入正式预报期的判定依据。

## 六、边界条件设置

### （一）上边界条件——流域面/栅格雨量、面/栅格潜在蒸散发过程

上边界条件是驱动流域产流、汇流计算的输入过程，须由地面监测站点观测数据经**地理空间插值方法**处理后得到面平均或栅格化过程，具体技术要求如下：

**1. 数据来源**

- 雨量数据：流域及周边自动雨量站（区域自动气象站、水文遥测站）逐时/逐日观测降雨；条件具备时可融合雷达估测降雨、卫星降水产品作为插值背景场或订正参考；
- 蒸散发数据：流域及周边气象站蒸发观测（蒸发皿/彭曼公式计算的潜在蒸散发ET0），气象要素（温度、湿度、风速、辐射）经彭曼-蒙特斯（Penman-Monteith）公式或彭曼公式计算逐日潜在蒸散发。

**2. 空间插值方法**

| 插值方法 | 适用场景 |
|---|---|
| 反距离权重法（IDW） | 站点分布相对均匀、地形影响较弱的流域，计算简便、稳定性好 |
| 普通克里金法（Ordinary Kriging） | 站点稀疏、需刻画空间相关结构的流域，可给出插值方差 |
| 协同克里金法（Co-Kriging，结合DEM高程） | 山区流域降雨/蒸散发存在明显地形梯度效应时采用 |
| 泰森多边形法（Thiessen Polygon） | 用于快速计算流域/子流域面平均雨量 |
| PRISM/梯度距离平方反比法 | 复杂地形、降雨随高程变化显著的流域 |
| 双线性/双三次样条插值 | 已有栅格化背景场（如雷达、卫星产品）与站点数据融合订正 |

**3. 输出成果**

- **流域面雨量/面潜在蒸散发过程**：按流域或子流域边界，将站点插值结果面积加权平均，得到逐流域（分区）面平均降雨、潜在蒸散发时间序列，供概念性水文模型（新安江模型、API模型、水箱模型）使用；
- **栅格雨量/栅格潜在蒸散发过程**：按统一栅格分辨率（如90m/250m/1km DEM网格）将插值结果栅格化，得到逐栅格降雨、潜在蒸散发时间序列，供分布式栅格水文模型使用；
- 15个流域分区应分别建立独立的插值计算方案（插值方法、站点组、栅格/分区边界矢量），并随站网变化（新增/停测站点）动态更新插值站点组合。

**4. 质量控制**

- 插值前对站点观测数据进行缺测识别、异常值剔除、时间对齐处理；
- 插值后与邻近参证站点、历史多年均值进行合理性校核，必要时进行降雨/蒸发量级订正。

### （二）内边界条件——上下游汇入流量过程

- 对于流域分区内部存在梯级子流域、区间支流汇入或水库（湖泊）调蓄节点的情形，须设置内边界条件，作为分区之间水量传递的输入过程；
- 内边界条件主要包括：**上游来水过程**（上游子流域/上游水文站实测或预报流量，作为下游分区模型的输入边界）、**区间支流汇入流量过程**（区间支流控制站实测流量或未控区间流量估算成果）、**水库出库（调节）流量过程**（水库实际调度出库流量或调度方案拟定流量）；
- 15个流域分区中涉及梯级衔接、区间入流或水库调节的分区（如流域五、流域十、流域十五等，见总览表备注），应分别明确内边界断面位置、数据来源及传递时间步长，实现分区间"接口对齐、逐时/逐日同步"的耦合计算。

### （三）下边界条件——流域出口流量过程

- 下边界条件为流域（分区）出口控制断面的流量过程，是模型模拟计算的下游边界，通常表现为：**流域出口实测流量过程**（用于模型模拟结果的验证与实时校正）、**下游衔接边界**（当该流域分区作为更大范围预报系统的上游输入时，其出口流量过程即作为下游河段/水动力模型的上边界条件）；
- 15个流域分区应分别在流域出口设置固定控制断面（原则上与现有水文站网重合），明确出口流量过程的观测/预报数据来源、时间分辨率及与实时校正模块的对接方式，保证预报成果可考核、可追溯、可衔接。

## 七、分流域（分区）配置台账模板

针对每一个流域分区，建议按以下模板逐一建立配置台账，实现"一流域一档"：

| 配置项 | 配置内容 | 数据来源/方法 |
|---|---|---|
| 流域编号/名称 | 例：流域01 / XX流域 | — |
| 模型结构 | 例：新安江模型（三水源） | 依流域气候、下垫面条件选定 |
| 核心初始参数 | K、WM、WUM/WLM/WDM、B、C、IM、SM、EX、KI、KG、CI、CG、CS、KE/XE等 | 历史洪水资料率定/相似流域移用 |
| 初始条件 | W0U、W0L、W0D（初始土壤含水量）、S0（初始自由水/地下水储量）、QG0（初始基流） | 前期实测退水过程反算/多年同期均值 |
| 预热期 | 起止时间、时长（天）、收敛判定阈值 | 按流域调蓄能力确定 |
| 上边界条件 | 面/栅格雨量、面/栅格潜在蒸散发过程；插值方法；站点组 | 雨量站、气象站观测数据地理空间插值 |
| 内边界条件 | 上下游汇入流量过程（上游来水、区间支流、水库出库） | 上游/支流水文站实测或水库调度数据 |
| 下边界条件 | 流域出口流量过程 | 出口控制站实测/预报流量 |

15个流域分区均应依据上述模板独立填报、独立存档，并纳入模型参数与边界条件统一管理平台，支持逐流域查询、修改、版本追溯及批量驱动预报运算。

## 八、小结

本方案围绕15个流域洪水预报模型分区，从"核心初始参数—初始条件—预热期—上/内/下边界条件"五个环节构建了完整的配置体系，重点明确了面/栅格雨量与面/栅格潜在蒸散发须由监测站点数据经地理空间插值方法（IDW、克里金、泰森多边形等）处理后作为模型驱动输入，实现了"参数分区独立设置、边界条件规范衔接、预报计算可追溯可核查"的技术要求，为后续15个流域洪水预报模型的建设、调试与业务运行提供统一、规范、可扩展的配置依据。
